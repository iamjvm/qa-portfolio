## 1. Scope

- Elements — Text Box, Buttons
- Forms — Student Registration Form

### Out of Scope:

- Interactions, Widgets, Book Store modules

- Mobile testing
- Automation

## 2. Test Approach

- Type: Manual Functional Testing
- Techniques: Equivalence Partitioning, Boundary Value Analysis,
Negative Testing, Exploratory Testing
- Bug Tracking: JIRA — component: DemoQA-Project2

## 4. Entry Criteria

- DemoQA site is accessible and loading correctly
- All 20 test cases are written and reviewed
- JIRA component DemoQA-Project2 is set up

---

## 5. Exit Criteria

- All 20 test cases executed
- All Critical and High bugs reported in JIRA
- Zero open Critical bugs remaining
- Test Summary Report completed

## 7. Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
| --- | --- | --- | --- |
| DemoQA site slow or unavailable | Low | High | Retest next session |
| Form fields change between sessions | Low | Medium | Screenshot all failures immediately |
| Unclear expected behaviour on some fields | Medium | Medium | Test both accept and reject — document either way |
